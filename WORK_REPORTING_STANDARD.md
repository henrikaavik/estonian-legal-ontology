# Work Reporting Standard v1 (team-wide)

Purpose: keep Henrik informed with clear, low-noise visibility.

## 1) Reporting must happen
- While active work is in progress: post a short progress update every ~15 minutes.
- If no active work is in progress: no periodic status messages.

## 2) Reporting must be meaningful
Each update must include:
1. Current concrete sub-step (what is being done right now).
2. Small progress delta since previous update (what changed).
3. Optional immediate next step.

Avoid: only project name, repeated identical text, or generic "still working".

## 3) Proper closure message is mandatory
When work finishes or pauses:
- send one short closure update immediately,
- include outcome in 1-3 bullets (what was done),
- mention next action only if relevant.

## Format (recommended)
- In-progress: "Now: <sub-step>. Progress: <delta>. Next: <optional>."
- Done: "Done/Paused: <task>. Completed: <bullet list>."
